MySql8.0

数据库权限操作

  • grant all privileges on . to ‘username’@’%’; 授予该用户所有操作权限
  • grant CREATE on . to ‘username’@’%’; 授予该用户所有创建数据库的操作权限
  • FLUSH PRIVILEGES; 刷新权限
  • select * from user; 查看权限(mysql 库)

    MYSQL权限详细分类：

  • 全局管理权限：
    1. FILE: 在MySQL服务器上读写文件。
    2. PROCESS: 显示或杀死属于其它用户的服务线程。
    3. RELOAD: 重载访问控制表，刷新日志等。
    4. SHUTDOWN: 关闭MySQL服务。
  • 数据库/数据表/数据列权限：
    1. ALTER: 修改已存在的数据表(例如增加/删除列)和索引。
    2. CREATE: 建立新的数据库或数据表。
    3. DELETE: 删除表的记录。
    4. DROP: 删除数据表或数据库。
    5. INDEX: 建立或删除索引。
    6. INSERT: 增加表的记录。
    7. SELECT: 显示/搜索表的记录。
    8. UPDATE: 修改表中已存在的记录。
  • 特别的权限：
    1. ALL: 允许做任何事(和root一样)。
    2. USAGE: 只允许登录–其它什么也不允许做。
